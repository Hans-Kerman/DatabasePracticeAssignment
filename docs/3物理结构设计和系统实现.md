# 二、物理设计

## 6 物理设计

在数据库逻辑结构设计完毕后，为了提高系统运行效率和查询性能，需要根据实际业务的查询预估，为关系模式设计合理的索引。本系统主要在以下几个维度建立索引：

### 1. 主键聚簇索引

- **目的**：聚簇索引决定了数据在磁盘上的物理存储顺序。
- **设计**：系统自动在所有表的主键上建立唯一索引，例如：
  - `reader(reader_id)`
  - `book(book_id)`
  - `book_copy(copy_id)`
- **作用**：保证实体完整性，并大幅提升基于编号的单条记录定位速度。

### 2. 外键非聚簇索引

- **目的**：系统属于典型的多表关联结构，例如借阅记录需要关联读者与单册，在多表 `JOIN` 操作时，外键上的索引能有效降低全表扫描开销。
- **设计**：在高频连接字段上建立非聚簇索引，例如：
  - `lend_record(reader_id)`
  - `lend_record(copy_id)`
  - `book_copy(book_id)`

### 3. 业务高频查询索引

- **目的**：针对读者端常用的条件检索优化。
- **设计**：
  - 在 `book(title)` 上建立 B-Tree 索引，应对日常频繁的图书名称检索。
  - 在 `book(isbn)` 上建立 B-Tree 索引，应对 ISBN 查询。
  - 在 `reader(card_number)` 上建立索引，加快刷卡/扫码登录时的身份核验。

## 7.1 完整性设计

### 1. 实体完整性

通过 `PRIMARY KEY` 约束保证每张表，例如读者表、图书表、借阅记录表等，都存在唯一标识。

### 2. 参照完整性

通过 `FOREIGN KEY` 约束保证数据流转一致性。例如，借阅记录表中的 `reader_id` 必须是读者表中已存在的实体，不能凭空捏造。

### 3. 用户自定义完整性

利用 `CHECK` 约束和 `DEFAULT` 默认值满足特定业务规则。例如：

- 馆藏单册物理状态只能取特定值：

```sql
CHECK (status IN ('在馆', '已借出', '遗失', '破损'))
```

- 评价星级限制在 1-5 之间：

```sql
CHECK (score >= 1 AND score <= 5)
```

- 读者账户状态：

```sql
CHECK (status IN ('正常', '冻结', '注销'))
```

## 7.2 表的创建

### 1. 图书书目表 `book`

```sql
CREATE TABLE book (
    book_id SERIAL PRIMARY KEY,
    isbn VARCHAR(20) UNIQUE NOT NULL,
    title VARCHAR(100) NOT NULL,
    author VARCHAR(100),
    publisher VARCHAR(100),
    pub_date DATE,
    price DECIMAL(8, 2)
);
```

### 2. 图书分类表 `category`

```sql
CREATE TABLE category (
    category_id SERIAL PRIMARY KEY,
    name VARCHAR(50) NOT NULL,
    description TEXT
);
```

### 3. 书目-分类映射表 `book_category`

```sql
CREATE TABLE book_category (
    book_id INT REFERENCES book(book_id) ON DELETE CASCADE,
    category_id INT REFERENCES category(category_id) ON DELETE CASCADE,
    PRIMARY KEY (book_id, category_id)
);
```

### 4. 读者信息表 `reader`

```sql
CREATE TABLE reader (
    reader_id SERIAL PRIMARY KEY,
    name VARCHAR(50) NOT NULL,
    card_number VARCHAR(20) UNIQUE NOT NULL,
    phone VARCHAR(20),
    valid_until DATE NOT NULL,
    status VARCHAR(10) DEFAULT '正常'
        CHECK (status IN ('正常', '冻结', '注销'))
);
```

### 5. 馆藏单册表 `book_copy`

```sql
CREATE TABLE book_copy (
    copy_id SERIAL PRIMARY KEY,
    book_id INT REFERENCES book(book_id) NOT NULL,
    location VARCHAR(50),
    status VARCHAR(10) DEFAULT '在馆'
        CHECK (status IN ('在馆', '已借出', '遗失', '破损')),
    create_time TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

### 6. 借阅记录表 `lend_record`

```sql
CREATE TABLE lend_record (
    lend_id SERIAL PRIMARY KEY,
    reader_id INT REFERENCES reader(reader_id) NOT NULL,
    copy_id INT REFERENCES book_copy(copy_id) NOT NULL,
    borrow_date DATE DEFAULT CURRENT_DATE,
    due_date DATE NOT NULL,
    return_date DATE,
    renew_count INT DEFAULT 0
);
```

### 7. 逾期罚单表 `penalty`

```sql
CREATE TABLE penalty (
    penalty_id SERIAL PRIMARY KEY,
    lend_id INT REFERENCES lend_record(lend_id) NOT NULL,
    reader_id INT REFERENCES reader(reader_id) NOT NULL,
    amount DECIMAL(8, 2) NOT NULL,
    status VARCHAR(10) DEFAULT '未缴清'
        CHECK (status IN ('未缴清', '已缴清')),
    pay_time TIMESTAMP
);
```

### 8. 预约排队表 `reservation`

```sql
CREATE TABLE reservation (
    res_id SERIAL PRIMARY KEY,
    reader_id INT REFERENCES reader(reader_id) NOT NULL,
    book_id INT REFERENCES book(book_id) NOT NULL,
    res_date TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    status VARCHAR(10) DEFAULT '排队中'
        CHECK (status IN ('排队中', '已通知', '已取消', '已完成'))
);
```

### 9. 图书评价表 `review`

```sql
CREATE TABLE review (
    review_id SERIAL PRIMARY KEY,
    reader_id INT REFERENCES reader(reader_id) NOT NULL,
    book_id INT REFERENCES book(book_id) NOT NULL,
    score INT CHECK (score >= 1 AND score <= 5),
    content TEXT,
    review_time TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

## 7.3 安全性设计

本系统通过基于角色的访问控制方案设计数据库安全性，主要划分为“图书管理员”与“普通读者”两个核心角色。

### 角色说明

- **Role_Admin 管理员角色**
  - 拥有整个业务系统的完整 CRUD 权限。
  - 负责日常业务办理和数据维护。

- **Role_Reader 读者角色**
  - 拥有书目查询的只读权限。
  - 拥有涉及自身业务的部分写入权限，例如：
    - 借阅查询
    - 发起预约
    - 发布评价
  - 禁止访问其他读者隐私数据。

### SQL 实现

```sql
-- 创建角色
CREATE ROLE role_admin;
CREATE ROLE role_reader;

-- 授予管理员角色所有表的操作权限
GRANT ALL PRIVILEGES ON ALL TABLES IN SCHEMA public TO role_admin;
GRANT ALL PRIVILEGES ON ALL SEQUENCES IN SCHEMA public TO role_admin;

-- 授予读者角色书目、分类的查询权限
GRANT SELECT ON book, category, book_category, book_copy TO role_reader;

-- 授予读者角色评价表、预约表的插入与查询权限
GRANT SELECT, INSERT ON review, reservation TO role_reader;

-- 实际应用中，创建具体用户并赋予角色
CREATE USER lib_admin1 WITH PASSWORD 'Admin@123';
GRANT role_admin TO lib_admin1;

CREATE USER stu_reader1 WITH PASSWORD 'Stu@123';
GRANT role_reader TO stu_reader1;
```

# 四、7.4 视图的创建

系统设计了以下 4 个核心视图，用于简化应用层复杂查询并保护基础表数据安全。

---

## 1. 综合图书库存视图 `v_book_inventory`

### 目的

方便读者或管理员直接查询一本书的“总拥有量”和“当前在馆可借量”。

### SQL

```sql
CREATE VIEW v_book_inventory AS
SELECT
    b.book_id,
    b.isbn,
    b.title,
    b.author,
    COUNT(c.copy_id) AS total_copies,
    SUM(CASE WHEN c.status = '在馆' THEN 1 ELSE 0 END) AS available_copies
FROM
    book b
LEFT JOIN
    book_copy c ON b.book_id = c.book_id
GROUP BY
    b.book_id, b.isbn, b.title, b.author;
```

---

## 2. 读者当前在借明细视图 `v_reader_borrowing`

### 目的

读者个人中心查看“我当前正在借哪些书，什么时候到期”。

### SQL

```sql
CREATE VIEW v_reader_borrowing AS
SELECT
    r.name AS reader_name,
    r.card_number,
    b.title AS book_title,
    c.location AS book_location,
    l.borrow_date,
    l.due_date,
    l.renew_count
FROM
    lend_record l
JOIN reader r ON l.reader_id = r.reader_id
JOIN book_copy c ON l.copy_id = c.copy_id
JOIN book b ON c.book_id = b.book_id
WHERE
    l.return_date IS NULL;
```

---

## 3. 超期违约警报视图 `v_overdue_alert`

### 目的

管理员每日盘点用，找出所有超期未还的记录及超期天数。

### SQL

```sql
CREATE VIEW v_overdue_alert AS
SELECT
    l.lend_id,
    r.name AS reader_name,
    r.phone,
    b.title AS book_title,
    l.due_date,
    CURRENT_DATE - l.due_date AS overdue_days
FROM
    lend_record l
JOIN reader r ON l.reader_id = r.reader_id
JOIN book_copy c ON l.copy_id = c.copy_id
JOIN book b ON c.book_id = b.book_id
WHERE
    l.return_date IS NULL
    AND l.due_date < CURRENT_DATE;
```

---

## 4. 图书口碑评价视图 `v_book_reviews`

### 目的

统计图书的平均评分和评论数，方便读者选书。

### SQL

```sql
CREATE VIEW v_book_reviews AS
SELECT
    b.book_id,
    b.title,
    COUNT(r.review_id) AS review_count,
    ROUND(AVG(r.score), 1) AS avg_score
FROM
    book b
LEFT JOIN
    review r ON b.book_id = r.book_id
GROUP BY
    b.book_id, b.title;
```

# 五、7.5 存储过程的创建

系统共创建 4 个存储过程。

---

## 1. 办理借书过程 `sp_borrow_book`

### 目的

输入读者 ID 和实体书 ID，执行借书核心逻辑，包含一系列状态检查。

### SQL

```sql
CREATE OR REPLACE PROCEDURE sp_borrow_book(p_reader_id INT, p_copy_id INT)
LANGUAGE plpgsql
AS $$
DECLARE
    v_reader_status VARCHAR(10);
    v_copy_status VARCHAR(10);
BEGIN
    -- 1. 检查读者状态
    SELECT status INTO v_reader_status
    FROM reader
    WHERE reader_id = p_reader_id;

    IF v_reader_status != '正常' THEN
        RAISE EXCEPTION '读者账户状态异常（%），无法借阅。', v_reader_status;
    END IF;

    -- 2. 检查单册状态
    SELECT status INTO v_copy_status
    FROM book_copy
    WHERE copy_id = p_copy_id;

    IF v_copy_status != '在馆' THEN
        RAISE EXCEPTION '该单册当前状态为（%），不可借阅。', v_copy_status;
    END IF;

    -- 3. 核心业务处理：插入借阅记录，默认借期30天
    INSERT INTO lend_record (reader_id, copy_id, borrow_date, due_date)
    VALUES (p_reader_id, p_copy_id, CURRENT_DATE, CURRENT_DATE + 30);

    -- 4. 更改单册物理状态
    UPDATE book_copy
    SET status = '已借出'
    WHERE copy_id = p_copy_id;

    COMMIT;
END;
$$;
```

## 2. 办理还书过程 `sp_return_book`

### 目的

根据借阅记录 ID 办理还书，更新实际归还时间，并恢复实体书状态为在馆。

### SQL

```sql
CREATE OR REPLACE PROCEDURE sp_return_book(p_lend_id INT)
LANGUAGE plpgsql
AS $$
DECLARE
    v_copy_id INT;
BEGIN
    -- 1. 查找对应的实体书ID
    SELECT copy_id INTO v_copy_id
    FROM lend_record
    WHERE lend_id = p_lend_id;

    IF NOT FOUND THEN
        RAISE EXCEPTION '未找到对应的借阅记录。';
    END IF;

    -- 2. 更新借阅表实际归还时间
    UPDATE lend_record
    SET return_date = CURRENT_DATE
    WHERE lend_id = p_lend_id;

    -- 3. 将实体书状态恢复为在馆
    UPDATE book_copy
    SET status = '在馆'
    WHERE copy_id = v_copy_id;

    COMMIT;
END;
$$;
```

---

## 3. 违约金清缴过程 `sp_pay_penalty`

### 目的

读者缴纳罚款后，核销罚单并检查是否可以解冻读者账户。

### SQL

```sql
CREATE OR REPLACE PROCEDURE sp_pay_penalty(p_penalty_id INT)
LANGUAGE plpgsql
AS $$
DECLARE
    v_reader_id INT;
    v_unpaid_count INT;
BEGIN
    -- 1. 更新罚单状态为已缴清
    UPDATE penalty
    SET status = '已缴清',
        pay_time = CURRENT_TIMESTAMP
    WHERE penalty_id = p_penalty_id
    RETURNING reader_id INTO v_reader_id;

    -- 2. 检查该读者是否还有其他未缴清的罚单
    SELECT COUNT(*) INTO v_unpaid_count
    FROM penalty
    WHERE reader_id = v_reader_id
      AND status = '未缴清';

    -- 3. 如果全部缴清，解冻账户
    IF v_unpaid_count = 0 THEN
        UPDATE reader
        SET status = '正常'
        WHERE reader_id = v_reader_id;
    END IF;

    COMMIT;
END;
$$;
```

---

## 4. 批量添加馆藏实体书过程 `sp_add_book_copies`

### 目的

采购新书后，给定书目 ID 和数量，自动循环生成指定数量的实体书记录。

### SQL

```sql
CREATE OR REPLACE PROCEDURE sp_add_book_copies(
    p_book_id INT,
    p_count INT,
    p_location VARCHAR
)
LANGUAGE plpgsql
AS $$
DECLARE
    i INT := 1;
BEGIN
    WHILE i <= p_count LOOP
        INSERT INTO book_copy (book_id, location, status)
        VALUES (p_book_id, p_location, '在馆');

        i := i + 1;
    END LOOP;

    COMMIT;
END;
$$;
```

# 六、7.6 触发器的创建

系统共创建 4 个触发器。

---

## 1. 自动生成超期罚单触发器 `trg_auto_penalty`

### 目的

在更新 `lend_record` 表，即还书动作时，如果实际归还时间大于应还时间，自动计算罚金并插入罚单表。示例规则为每天 0.5 元。

### SQL

```sql
CREATE OR REPLACE FUNCTION trg_func_auto_penalty()
RETURNS TRIGGER AS $$
DECLARE
    v_days INT;
BEGIN
    -- 仅在更新了归还时间且超期的情况下触发
    IF NEW.return_date IS NOT NULL
       AND NEW.return_date > NEW.due_date THEN

        v_days := NEW.return_date - NEW.due_date;

        INSERT INTO penalty (lend_id, reader_id, amount)
        VALUES (NEW.lend_id, NEW.reader_id, v_days * 0.5);
    END IF;

    RETURN NEW;
END;
$$ LANGUAGE plpgsql;

CREATE TRIGGER trg_auto_penalty
AFTER UPDATE ON lend_record
FOR EACH ROW
EXECUTE FUNCTION trg_func_auto_penalty();
```

---

## 2. 自动冻结违约读者触发器 `trg_freeze_reader`

### 目的

当 `penalty` 罚单表被插入新的“未缴清”罚单时，自动将读者状态修改为“冻结”，限制其后续借阅。

### SQL

```sql
CREATE OR REPLACE FUNCTION trg_func_freeze_reader()
RETURNS TRIGGER AS $$
BEGIN
    IF NEW.status = '未缴清' THEN
        UPDATE reader
        SET status = '冻结'
        WHERE reader_id = NEW.reader_id;
    END IF;

    RETURN NEW;
END;
$$ LANGUAGE plpgsql;

CREATE TRIGGER trg_freeze_reader
AFTER INSERT ON penalty
FOR EACH ROW
EXECUTE FUNCTION trg_func_freeze_reader();
```

---

## 3. 限制最大借阅量触发器 `trg_check_borrow_limit`

### 目的

在进行借阅操作，即插入 `lend_record` 之前，检查该读者未归还的书本数量。如果已达 5 本，则抛出异常阻止借阅。

### SQL

```sql
CREATE OR REPLACE FUNCTION trg_func_check_borrow_limit()
RETURNS TRIGGER AS $$
DECLARE
    v_active_borrows INT;
BEGIN
    SELECT COUNT(*) INTO v_active_borrows
    FROM lend_record
    WHERE reader_id = NEW.reader_id
      AND return_date IS NULL;

    IF v_active_borrows >= 5 THEN
        RAISE EXCEPTION '该读者在借图书已达5本上限，请先归还！';
    END IF;

    RETURN NEW;
END;
$$ LANGUAGE plpgsql;

CREATE TRIGGER trg_check_borrow_limit
BEFORE INSERT ON lend_record
FOR EACH ROW
EXECUTE FUNCTION trg_func_check_borrow_limit();
```

## 4. 评价权限校验触发器 `trg_check_review_auth`

### 目的

校验是否是真实读者可以进行评价，防止水军刷好评。在向 `review` 表插入评价前，校验该读者是否真的借阅过这本书，即在借阅记录表中存在过。

### SQL

```sql
CREATE OR REPLACE FUNCTION trg_func_check_review_auth()
RETURNS TRIGGER AS $$
DECLARE
    v_has_borrowed INT;
BEGIN
    -- 联合单册表查询读者是否借过对应的书目
    SELECT COUNT(*) INTO v_has_borrowed
    FROM lend_record l
    JOIN book_copy c ON l.copy_id = c.copy_id
    WHERE l.reader_id = NEW.reader_id
      AND c.book_id = NEW.book_id;

    IF v_has_borrowed = 0 THEN
        RAISE EXCEPTION '操作拒绝：您未曾借阅过该图书，无法发表评价。';
    END IF;

    RETURN NEW;
END;
$$ LANGUAGE plpgsql;

CREATE TRIGGER trg_check_review_auth
BEFORE INSERT ON review
FOR EACH ROW
EXECUTE FUNCTION trg_func_check_review_auth();
```

# 八、对前两阶段文档的实现落地补充

本阶段将前两阶段的概念/逻辑设计具体落地为 PostgreSQL 实现：

1. **书目与单册分离已落地**
   - `book` 存储抽象书目信息。
   - `book_copy` 存储实体单册信息。
   - `book_copy.book_id` 关联 `book.book_id`。

2. **多分类已落地**
   - `book_category` 使用复合主键：
     - `(book_id, category_id)`
   - 两个外键均设置 `ON DELETE CASCADE`。

3. **借还业务已落地**
   - `sp_borrow_book` 插入借阅记录，并更新单册状态为 `已借出`。
   - `sp_return_book` 更新归还日期，并恢复单册状态为 `在馆`。

4. **逾期罚单已落地**
   - `trg_auto_penalty` 在 `lend_record` 更新后检测是否逾期。
   - 逾期时按 `v_days * 0.5` 生成罚单。

5. **读者冻结/解冻已落地**
   - 新增未缴清罚单后，`trg_freeze_reader` 自动冻结读者。
   - 缴清罚单后，`sp_pay_penalty` 检查是否仍有未缴清罚单；若无，则恢复为 `正常`。

6. **评价权限已落地**
   - `trg_check_review_auth` 校验读者是否借阅过对应书目。
   - 未借阅过则拒绝插入评价。

7. **读者最大借阅量已落地**
   - `trg_check_borrow_limit` 限制未归还借阅记录数量最多为 5。
